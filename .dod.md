# Definition of Done (DoD) - Quisy-TTS

This document defines the quality standards and requirements that must be met before a task or feature is considered complete in this repository.

## 1. Code Quality & Formatting
- **Formatting**: The code must be formatted using `ruff format`. Run `uv run ruff format .` before committing.
- **Linting**: The code must pass all linting rules defined in `pyproject.toml` using `ruff`. Run `uv run ruff check .`.
- **Type Checking**: The code must pass static type checking with `mypy`. Run `uv run mypy .`.

## 2. Testing
- **Test Coverage**: All new features and bug fixes must be accompanied by corresponding unit or integration tests.
- **Passing Tests**: The entire test suite must pass. Run `uv run pytest`.
- **No Regressions**: Existing tests must continue to pass.

## 3. Documentation
- **API Docs**: If API changes are made, documentation in `README.md` and/or `docs/` must be updated to reflect the changes.
- **Changelog**: All user-facing changes (features, bug fixes, breaking changes) must be documented in `CHANGELOG.md`.

## 4. Workflow
- **Commit Standards**: Commit messages should be concise and descriptive.
- **Branching**: All work must be performed on topic branches.
- **PR**: If applicable, a Pull Request must be created, reviewed, and approved before merging into `main`.

## 5. Security & Safety
- **Secrets**: No secrets or `.env` files with credentials should be committed.
- **Dependencies**: New dependencies must be added to `pyproject.toml` and verified.
